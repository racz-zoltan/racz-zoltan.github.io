CarryPass User Manual

A practical guide to help you get started with your secure, offline CarryPass experience.

1. Introduction

CarryPass is a privacy-first credential manager designed for offline use. It lets teams securely share credentials through end-to-end encrypted vaults, with QR-based device enrollment and per-member access control — no cloud, no third party involved. It also includes a deterministic password generator: a tool that calculates strong passwords on demand without ever storing them, which comes in especially handy for generating the high-entropy passwords the vault requires.

2. Getting Started

CarryPass is a Progressive Web App (PWA), which means you can install it like a native app — no tracking, no account, and full offline support.

Install on Desktop

Open CarryPass in a compatible browser such as Chrome or Edge. Look for the install icon in the address bar, then click it to install.

Optionally, pin it to your taskbar. You can now launch it like a regular app — even offline.

Install on Mobile

In Chrome, Safari, or Firefox, open the CarryPass website and tap the Share icon (a square with an upward arrow).

Then tap "Add to Home Screen" and confirm with "Add". CarryPass now behaves like a native app — offline and just a tap away.

  • No account required: You are never tracked.
  • No cloud storage: All data stays on your device.
  • Fully offline: Works without internet after install.
  • Browser support: Chrome, Edge, Firefox, Safari.

3. CarryPass Login and Adaptive Icons

CarryPass login uses two things you can reproduce without any server: your master passphrase and an adaptive icon path. The app does not store your master secret. Instead, it rebuilds the same cryptographic key locally when you type the same passphrase and complete the same icon path.

3.1 How the Icon Path Works

After your passphrase reaches the required strength, CarryPass shows an icon grid automatically. You select icons by identity, not by position. The positions are randomized each session, so you should remember the icons themselves, not where they appeared.

The current profile uses three grids. On each grid, select four icons in order. After every icon selection CarryPass performs a memory-hard Argon2id calculation and uses the resulting state to derive the next step. When a grid is complete, the next grid appears automatically.

No extra recovery seed: the icon path remains stateless and deterministic. If you know your master passphrase and remember the same icon path, you can reproduce the same keys on another device.

3.2 Registration Practice

During registration you complete the icon path three times:

  • First selection: choose your icon path.
  • Guided practice: the correct icons briefly pulse to help you learn them.
  • Final confirmation: repeat the path without assistance.

This teaching phase helps you remember icon identity instead of screen position.

3.3 Login

On normal login, type your master passphrase and select the same adaptive icon path. CarryPass may take several seconds because Argon2id is intentionally memory-hard. The selected icon pulses while the calculation is running.

Important: if you forget your master passphrase or adaptive icon path, CarryPass cannot recover it. There is no account, server-side reset, or backdoor.

4. Team Vaults

The TEAMS tab gives access to encrypted credential vaults that an admin has set up and distributed to team members. Each member can only see the credentials assigned to them.

All vault operations use end-to-end encryption (E2EE). The server and admin never see member passwords, and the file's encryption is tied to the device's enrollment — even a leaked vault file cannot be decrypted without the trusted device's secret.

3.1 Member Access — Step by Step

To access your team vault for the first time you need to complete a one-time device enrollment step.

1

Scan your onboarding QR code with CarryPass. Use the Scan button to open the camera, or the Browse button to upload a QR image your admin provided. This enrolls your device as the trusted device and stores your encrypted member identifier and TOTP secret locally.

2

Enter your Member Access Password in the password field. This is the password you chose at member finalization.

3

Click Access Vault. If a vault file is not preloaded, use the Browse button next to it to upload the .json vault file your admin gave you.

You now have access to your assigned credentials. Each entry shows the service name, username, the notes, and a password you can reveal and copy.

How decryption works: Your member section is encrypted with a key derived from your password combined with the TOTP secret stored on your trusted device. Both halves are required. This is why the QR enrollment is essential — without it, even a correct password cannot decrypt the vault. There is no separate 6-digit code to type at unlock; possession of the trusted device is what proves your identity cryptographically.
Trusted device indicator: A Trusted Device indicator appears at the top of the screen when your device is enrolled. On subsequent visits you only need your Member Access Password — the QR scan is one-time.

3.2 Sharing Trust with Another Device

If you want to use your vault on a second device, click the button next to the Trusted Device indicator. This generates a QR code you can scan with CarryPass on the other device to transfer trust — without involving your admin.

Alternatively, you can export your settings (menu → Export Settings) on your current device and import them on a new one. The exported file contains your trusted-device state, encrypted under your master password and adaptive icon path — meaning the new device must complete CarryPass login first before importing. This serves as a self-recovery path if you lose access to your original device.

3.3 Vault Expiry

Vaults can have an expiry date set by the admin. There are two kinds of expiry to be aware of:

  • Vault expiry — the global expiry shown at the top of the panel. This is informational only — it reminds the admin to refresh the vault, but it does not block your access.
  • Your access expiry — your individual access end date, set by the admin. After this date you will not be able to open the vault as a member, and you will need to ask the admin to extend your access.

3.4 Admin Vault

An admin is simply a member who has been flagged as the vault's administrator. There is only ever one admin per vault. To enter the admin panel, the admin uses the Unlock Admin Vault entry and provides their member password — the same password they would use as a regular member. There is no separate "admin password."

From the admin panel the admin can:

  • Create teams — logical groups for organising members and credentials
  • Create members — each member gets a unique ID and a TOTP secret generated at finalization
  • Assign members to teams — controls which credentials each member can access
  • Add credentials — service name, URL, username, password, and notes
  • Finalize members — the member types their password, the system generates the onboarding QR code
  • Promote a member to admin — toggle the Admin flag on a member to designate them as the new admin
  • Export the vault — produces the encrypted .json file to deploy or distribute
  • Export the audit log — a record of admin actions during the current session
Important: The admin's role gives them tools to manage the vault, but the cryptographic protections are uniform — every member's section is encrypted with their own combined password+TOTP key.

3.5 Creating a Member — Admin Workflow

1

In the admin panel, go to Members → click Create New Member.

2

Enter a member name.

3

Click Create Member. The member appears in the list as Pending.

Optionally set an access expiry date. Toggle the Admin flag if you intend to promote this member to admin.

4

Click Finalize next to the pending member's name. The member types their own password on your device. The system generates a fresh TOTP secret, derives the member's encryption key from password and TOTP combined, and produces an onboarding QR code.

5

Handle the QR code carefully — see below.

6

Assign the member to one or more teams via the Assignments section.

7

Click Export Vault and deploy or distribute the .json file. Each export rotates all team keys, so previous vault files held by removed members will not match the new file.

Critical: the QR code is the only enrollment material. The QR encodes the member's TOTP secret — without it, no device can decrypt the vault for that member, and CarryPass has no recovery mechanism. The recommended handling:
  • Best: Have the member scan the QR with CarryPass on their own device (e.g. their phone) immediately during the finalization session. Their device becomes a trusted device on the spot, and you delete the QR from your device when done.
  • Also do: The member should make a hard copy (print, photograph onto offline storage, or save to a secure offline location) before deletion. This is their recovery medium if they lose their device.
  • If the member can't scan immediately: Print the QR or transfer the image to them through a secure channel, then delete it from your device. Encourage them to keep the hard copy and not store it on the same device they'll be using.
Admins must keep their own hard copy. If you lose access to your trusted device and have no QR backup, you cannot recover — the vault becomes inaccessible to everyone. Print or otherwise preserve your own onboarding QR before deleting it from your device. Treat it like any other recovery key: store it offline, somewhere only you can access.
Bootstrap: When CarryPass opens with no existing vault, the admin entry presents an empty admin panel directly. Create yourself as the first member and finalize. Grants admin status to the finalized member, ensuring the vault always has a designated admin.

5. Password Generator

The PASSWORDS tab uses a deterministic model — the same inputs always produce the same password, so nothing needs to be stored or synced. You can regenerate any password on any device, even offline. This is particularly useful for generating the strong passwords that vault access requires.

How It's Secured

  • Argon2id — memory-hard hashing that resists brute-force attacks, run twice (login + per-credential)
  • HKDF-SHA256 — domain-separated keystream expansion
  • Rejection sampling — unbiased character selection from the keystream
  • Everything runs in your browser — nothing is sent anywhere

What You Control

  • Service — the name or URL of the site (e.g. "github.com")
  • Password type / settings — choose a template or set manually
  • Variant Secret Password — your master password, never stored
  • Variant — a counter that lets you regenerate alternative passwords from the same inputs

4.1 Identity Mode — Recommended for everyday use

When Identity mode is on (the default), the generator uses password templates. Each template bundles all the technical settings — character set, length, and variant — into a single named choice. This removes the mental load of configuring settings individually: you simply enter the site name, pick a template, and get a strong password.

Built-in templates include:

  • Structured — mixed characters with a separator, high entropy
  • No symbols — letters and numbers only, for sites that reject symbols
  • Short — compact output for length-restricted sites
  • Mixed — varied character distribution
  • PIN 4 / PIN 6 — numeric only, for device or bank PINs
  • Long — maximum length for high-security accounts
  • Passphrase (EFF Long/Short/Memorable) — word-based passphrases from the EFF word lists, easy to remember and type

The Password variant selector lets you pick between several pre-derived alternatives for the same template — useful if a site forces a password change or has a conflicting requirement. This is a quick rotation mechanism that doesn't require regenerating from scratch.

Custom templates: If none of the presets match a site's requirements, you can create your own template via the menu → Create custom type. Set the character types, length, and variant number once — then reuse it by name across any service, just like the built-in options.

4.2 Manual Mode — Generating passwords without your account secrets

When Identity mode is off, all settings are exposed individually: character toggles, length, and the variant field. You also provide a Variant Secret Password that is independent of your CarryPass master password.

The primary purpose of manual mode is generating passwords whose cryptographic input is not your CarryPass identity. The most common scenario:

  • Shared device scenarios — A team member generates their own vault access password on the admin's device using their own secret, then regenerates the same password on their own device later with identical inputs. The admin never sees the member's secret, and the password they generate is bound to the member's typed input rather than the admin's CarryPass identity.
  • One-time use — When you need a quick password without saving anything to your CarryPass profile.
  • Fine-grained control — When a specific combination of settings is needed that doesn't fit any template.
Because the Variant Secret Password in manual mode is separate from your CarryPass master password, the person on whose device the generation happens never learns the other person's secret. The password derived in manual mode depends only on what is typed into the manual-mode fields.

4.3 The Variant Field

The Variant field accepts numbers from 1 to 99999. Changing this number produces a completely different password from the same other inputs. Its purpose is to let you generate alternative passwords without changing anything else — useful when you need to rotate a password but want to keep the same template, length, and character settings.

For example: if you need to change your GitHub password because of a forced rotation, you can keep all your other inputs identical and just increment the Variant. The new password is unrelated to the old one, but everything else is preserved.

The value is hidden by default — click Show next to it to reveal it. Note the variant number you used per service, because to regenerate the same password later you need to provide the same number. Treat it as part of the input, not as a security setting.

The Variant field is just a counter. It is part of the input: changing it changes the generated output, but it is not a security setting by itself. The actual cryptographic strength of generated passwords primarily depends on the strength of your Master Password, plus the password length and character set you choose.

4.4 Saving Settings Locally (Identity Mode Only)

After generating a password in Identity mode, click Save locally in the result card. This saves your settings (service name, template or character configuration, variant) — not the password itself — encrypted in your browser's local storage.

Saved services appear in the panel on the left. Click any saved service to instantly reload its settings and regenerate the password.

Saved settings are not available in Manual mode. The Variant Secret Password used in manual mode is a separate secret per use — typed in for the moment, then forgotten. Saving the settings would require either saving the password itself (which CarryPass doesn't do) or saving useless settings without the secret needed to reproduce the output. Manual mode is meant for ephemeral, secret-on-demand workflows; identity mode is what supports the saved-services feature.

You can export your saved settings as an encrypted file (via the menu ) and import them on another device. The exported file also includes your trusted-device state for team vaults, providing a self-recovery path if you lose access to your device.

Because saved settings are stored in your browser’s local storage, you should export them regularly if you rely on them. Local browser data can be lost if you clear site data, reset the browser, reinstall the app, use a cleanup tool, or if the browser removes stored data automatically under storage pressure. CarryPass does not store a server-side backup, so an encrypted export file is the recommended way to keep a recoverable copy of your saved settings and trusted-device state.

4.5 Phishing Resistance via Saved Services

Because passwords are deterministically derived from the service name you type, even a one-character difference produces a completely different password. CarryPass adds a visible match indicator to convert this property into phishing protection:

  • A green indicator means the service name or address you entered — whether typed manually or pasted from the browser address bar — matches a previously saved service exactly.
  • A red indicator means the typed name does not match any saved service — possibly because it's a typo, possibly because it's a phishing lookalike.

The indicator is informational, not blocking — but it gives you a chance to verify the URL before generating credentials for what might be a fake site.

4.6 Screen Lock

You can set a screen lock code via the menu → Set Screen Lock. The code must be at least 6 characters. After a period of inactivity the app locks itself and requires your screen lock code to resume. This protects against casual access on a shared device without requiring you to re-enter your master password and adaptive icon path.

The screen lock is a UI convenience, not a strong cryptographic boundary. Three wrong attempts wipe the in-memory state and force full re-authentication. The full-session inactivity timeout provides an additional limit on how long a locked screen can remain attackable. By default, this timeout is 30 minutes, but users can change it in the security settings.

6. Optional: Encrypted Local Storage

You may choose to store your password settings (not the password itself) in encrypted localStorage for convenience.

  • This feature remembers your preferred settings per service in identity mode.
  • The data is encrypted locally with a session key derived from your master password and adaptive icon path.
  • You can delete this data at any time from within the app.
  • You can optionally export the encrypted settings and import them on another device — useful for migrating to a new device.

7. Security Tips

  • Use a strong master passphrase — at least 16 characters, ideally a word-based passphrase. CarryPass offers a Diceware suggestion at registration to help.
  • Use a consistent and unambiguous service name format (e.g. always github.com, never GitHub).
  • Never share your master passphrase or your member access password with anyone.
  • Keep a hard copy of your team-vault onboarding QR code in a secure offline location. This is your only recovery path if your device is lost.
  • Clear local storage if switching or handing over your device.
  • Keep your trusted device under your control — it holds the TOTP secret that, combined with your password, decrypts your vault section.
  • Back up your settings periodically (menu → Export Settings) so you can recover access if you lose your device.

8. Frequently Asked Questions

How do I access my vault on a new or second device?

For a second device you already use: click the button next to the Trusted Device indicator to generate a transfer QR code. Scan it with CarryPass on the new device — no admin involvement needed.

For a completely new device (e.g. after a phone replacement): if you previously exported your settings, you can import the file on the new device after completing CarryPass login. If you have your hard-copy onboarding QR, you can scan it on the new device. If neither is available, ask your admin to re-issue your QR code.

Why is there no 6-digit TOTP code to type?

CarryPass uses your TOTP secret directly as a cryptographic key factor, not as a typed code. The secret stored on your trusted device combines with your password to derive the decryption key. You don't need an authenticator app for vault access — possession of the trusted device is what proves your identity. This makes login simpler while making the underlying cryptography stronger: an attacker who has the vault file but not your device cannot decrypt anything regardless of how well they guess your password.

Can the admin see my credentials or passwords?

No. Each member's section is encrypted with a key derived from their own password + TOTP secret. Your member password is only known to you.

What's the difference between vault expiry and access expiry?

The vault expiry is informational — it reminds the admin that the vault should be refreshed. It does not block your access. Your personal access expiry is set by the admin and does block your access after the date passes.

Do I need internet to use CarryPass?

No. CarryPass is designed to work fully offline once installed as a PWA. The vault file is cached locally, all cryptographic operations run in your browser, and the password generator never contacts any server. An internet connection is only needed for the initial install and to receive a new vault file from your admin.

What if I forget my Member Access Password?

There is no recovery mechanism — it is never stored anywhere. Contact your admin, who can re-finalize your member entry with a new password and re-issue your onboarding QR code.

What if I forget my master passphrase for the password generator?

There is no way to recover it — CarryPass never stores it. If you remember all the same inputs (service name, master passphrase, length, variant, character settings), the same password will always be regenerated. If the master passphrase is lost, those generated passwords cannot be recovered.

Why does a small change in the service name produce a completely different password?

The service name is part of the cryptographic input. Even a single character difference — a capital letter, a trailing slash, a space — produces an entirely different output. This is by design: it provides phishing resistance, since a lookalike domain (paypa1.com vs paypal.com) generates a different password that won't work on the real site. Use a consistent format from the start, such as always using github.com rather than alternating with GitHub.

Can multiple teams share the same vault?

Yes — this is the intended use. A single vault can contain multiple teams, each with their own credentials and members. Each member only sees what they have been assigned to. The admin manages all teams from one place and exports a single vault file.

What is the "Admin" toggle on a member?

The Admin toggle designates which member is the vault's administrator. Exactly one member can have this flag at any time. Toggling another member to admin (and clearing your own flag) performs an admin handover — after the next export, the new admin can manage the vault. Most members never need to think about this toggle; it is only used during initial setup or admin handover.

What is the Variant field for?

The Variant is a number (1–99999) that lets you generate alternative passwords from the same inputs. If a service forces you to change your password, you can increment the Variant and keep all other inputs the same — the new password will be unrelated to the old one, but everything else stays consistent. Note the value you used per service, because regenerating the same password requires the same Variant.

Can I save settings in Manual mode?

No. Saved-settings only works in Identity mode. Manual mode uses a separately-typed Variant Secret Password that's not derived from your CarryPass identity, so the saved settings would be useless without the secret to reproduce the output. Manual mode is designed for ephemeral, on-demand workflows — typically when generating a password whose cryptographic input shouldn't be your CarryPass masterpassphrase.
