Read a use case

I want to log into the https://quotes.toscrape.com site, where I registered earlier with the password created with CarryPass. In the Website name field I type 'Quotes', then I type my Master key 'Password123', click on My Salt and the value changes to 'VKHRT2YZTZDWY3NULYTLKTSRWKYTT2ZB'. After that I click on the Create a [xX-7x] Password button. CarryPass creates the password 'txCC1qz-yHX5dpF-yfekGgK' and copies it to the clipboard so I can paste it in the password field on the login page of 'Quotes'. As long as the parameters are the same I always get back the same password, and while I do not have to remember its complexity, it stays private using CarryPass. In order to create a cryptographycally more secure password, I increase the Iteration count and use my own Salt. For some sites I generate a random salt that I keep secret. CarryPass does not store passwords or any data generated or entered on this site.
